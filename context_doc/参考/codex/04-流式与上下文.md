### 事件处理 (stream_events_utils.rs)
- `handle_non_tool_response_item()` - 处理文本/消息项
- `handle_output_item_done()` - 完成项
- `record_completed_response_item()` - 持久化到历史

### 关键特点
- **零缓冲**: 每个 chunk 立即处理并发送给客户端
- **真正的流式管线**: API → 解析 → 事件发射 → UI 渲染，全程流式

```rust
    history: Vec<TurnItem>,                     // 对话历史

impl ContextManager {

    async fn clone_history() -> Vec<ResponseItem> { ... }

    // 记录新项到对话中
    async fn record_items(&mut self, items: &[ResponseItem]) { ... }

```

### 自动压缩
当 token 使用达到阈值时：
1. 保存当前轮次状态
2. 总结早期轮次
3. 用压缩摘要替换
4. 继续轮次（释放了 token 空间）

## 任务系统 (tasks/)

所有任务实现 `SessionTask` trait：
```rust
pub(crate) trait SessionTask: Send + Sync + 'static {
    fn kind(&self) -> TaskKind;
    async fn run(self: Arc<Self>, session, ctx, input, cancel_token) -> Option<String>;
    async fn abort(&self, session, ctx) { }
}
```

### 任务类型
| 任务 | 用途 |
|------|------|
| `RegularTask` | 正常用户轮次执行 |
| `CompactTask` | 压缩对话历史 |
| `ReviewTask` | 审查待定变更 |
| `GhostSnapshotTask` | 无用户轮次的状态快照 |
| `UndoTask` | 撤销上次操作 |
| `UserShellCommandTask` | 直接执行用户 Shell 命令 |

## 配置系统

- **格式**: TOML
- **位置**: `~/.codex/config.toml` 或项目根目录
- **JSON Schema**: `codex-rs/core/config.schema.json`

### 配置层级
1. 代码内置默认值
2. 用户 TOML 配置文件
3. 每会话覆盖
4. 运行时 Feature Flags

## Rust 关键模式

| 模式 | 用法 |
|------|------|
| `Arc + Tokio Mutex` | 异步共享状态 |
| `Sender/Receiver Channel` | 事件流 |
| `CancellationToken` | 优雅中断 |
| `TurnContext` (Scoped) | 轮次生命周期管理 |
| `Trait Objects` | 多态工具处理 |
| `AsyncGenerator` | 流式响应处理 |
| `SessionTask trait` | 可组合的任务系统 |

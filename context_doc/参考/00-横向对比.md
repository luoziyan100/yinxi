## 架构要素对比

### 1. Agent 循环

| 项目 | 循环模式 | 特点 |
|------|---------|------|
| **Codex** | `run_turn()` 无限循环 + Tokio async | Turn-based 状态机，CancellationToken 中断 |
| **Claude Code** | 核心二进制（未开源）| 通过插件扩展行为 |
| **Pi-mono** | `AgentSession` 事件队列 | 事件串行处理，Promise 链保证顺序 |
| **Learn CC** | `while tool_use: execute → loop` | 最简循环，41 行起步 |
| **Yinxi (我们)** | `runLoop()` while 循环 | 流式 + 工具执行 + 截断 |

### 2. 工具系统

| 项目 | 定义方式 | 数量 | 特色 |
|------|---------|------|------|
| **Codex** | Rust trait 实现 | 20+ | 审批+沙箱管线、并行执行 |
| **Claude Code** | 内置 + 插件扩展 | 15+ | 工具限制模式、权限层级 |
| **Pi-mono** | `AgentTool` 接口 | 7 内置 + 扩展 | 扩展可包装/拦截工具 |
| **Learn CC** | Python dict dispatch | 4-10 | 从 1 个(bash) 到完整套件 |
| **Yinxi (我们)** | `Tool` 接口 | 6 | Read/Write/Edit/Bash/Glob/Grep |

### 3. 流式处理

| 项目 | 方式 | 特点 |
|------|------|------|
| **Codex** | WebSocket/SSE → Channel | 零缓冲、真流式 |
| **Pi-mono** | Agent 事件 → 监听器 | message_start/update/end 三阶段 |
| **Yinxi (我们)** | AsyncGenerator → EventListener | text_delta/tool_use_start/tool_result |

### 4. 上下文管理

| 项目 | 策略 | 触发 |
|------|------|------|
| **Codex** | 自动压缩 + 摘要 | token 达到阈值 |
| **Pi-mono** | LLM 摘要 + 溢出恢复 | 85% 容量 / 溢出错误 |
| **Learn CC** | 三层压缩 (micro/auto/archival) | 80% 容量 |
| **Yinxi (我们)** | 中间消息移除 + 工具结果截断 | 85% 容量 |

### 5. 会话持久化

| 项目 | 格式 | 特色 |
|------|------|------|
| **Codex** | 内存 + WebSocket 缓存 | response_id 链式引用 |
| **Pi-mono** | JSONL 文件 | 树形结构（id/parentId）、分支、标签 |
| **Learn CC** | .tasks/ JSON 文件 | 依赖图、外部状态 |
| **Yinxi (我们)** | 内存数组 | 无持久化（待实现） |

### 6. 扩展/插件

| 项目 | 机制 | 能力 |
|------|------|------|
| **Codex** | MCP 服务器 | 外部工具集成 |
| **Claude Code** | 命令 + Agent + 技能 + Hooks + MCP | 全方位可扩展 |
| **Pi-mono** | Extension API + 工具包装 | 工具拦截、自定义命令、UI widget |
| **Learn CC** | 技能文件 (SKILL.md) | 按需知识注入 |
| **Yinxi (我们)** | 无 | 待实现 |

## 关键设计模式汇总

### 所有 Agent 共有
1. **循环**: while (has_tool_calls) → execute → feedback → loop
2. **工具定义**: JSON Schema 描述参数
3. **流式**: 异步生成器/事件流
4. **系统提示**: 动态构建（环境 + 工具描述 + 项目上下文）

### 生产级 Agent 特有
1. **审批/沙箱**: 危险操作前确认 (Codex 最完善)
2. **上下文压缩**: LLM 摘要 vs 消息移除 (Pi-mono、Learn CC)
3. **会话持久化**: 跨重启恢复 (Pi-mono: JSONL 树)
4. **并行工具/Agent**: Promise.all / FuturesOrdered (Codex、Claude Code)
5. **扩展系统**: 插件/Extensions/Hooks (Claude Code、Pi-mono)
6. **多模型支持**: 动态切换模型和推理级别

### 教学项目展示的核心洞见
1. **最小 Agent = 1 个循环 + 1 个工具** (Learn CC s01)
2. **Runnable 是万物之基** (AI Agents From Scratch)
3. **按需加载 > 全量注入** (Learn CC s05, Claude Code Skills)
4. **外部状态 > 对话内状态** (Learn CC s07)
5. **ReAct = Thought + Action + Observation** (AI Agents From Scratch)
